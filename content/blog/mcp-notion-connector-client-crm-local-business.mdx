Your best clients walk through the door every six weeks. They always order the same thing, prefer the same stylist, and mentioned once that they hate when you're running late. The problem? That information lives in your head — and when you're slammed on a Tuesday afternoon, there's no way to look it up, share it with your team, or use it to book them again before they try the competitor down the street.

Most small business owners know they need a CRM. Most also never use one. The reasons are always the same: too complex to set up, requires constant manual data entry, built for enterprise sales teams not a 3-chair barbershop, and costs $75/month for features you'll use 10% of.

This guide changes that equation entirely. You're going to build a self-updating CRM using **Notion** (free) and **Claude's Model Context Protocol (MCP)** — a system where AI automatically logs client visits, preferences, follow-up dates, and revenue data without you typing a single field manually.

## Why Repeat Clients Are Worth Building a System For

Before we get into the tech, let's talk numbers. Research consistently shows that **repeat customers spend 67% more per visit than new customers** and cost five times less to retain than to acquire. For a local business running on referrals and word-of-mouth, your top 20% of clients likely generate 60–80% of your revenue.

But most local businesses treat every visit like it's the first one. No notes from last time. No record of what services they liked. No system to follow up two days after a highlight job to ask how it's settling. No birthday message. No "it's been 10 weeks, time to book your next grooming appointment" nudge.

A proper CRM solves all of this — but only if it's actually being updated. Manual data entry doesn't survive contact with a real workday. The solution is to make the CRM update itself.

## What You're Building

The stack is intentionally minimal:

- **Notion** (free plan works) — your CRM database
- **Notion MCP server** — the bridge that lets Claude read and write to Notion
- **Claude API** (claude-sonnet-4-5 or better) — the AI brain that processes appointment data and updates records
- **A simple Python webhook** — receives appointment events from your booking software and triggers the pipeline

When a client appointment completes, your booking software (Vagaro, Acuity, Square Appointments, or even a manual trigger) sends a webhook to your Python script. Claude reads the appointment notes, finds the client record in Notion, and updates every relevant field — automatically.

## Setting Up Your Notion CRM Database

Start by creating a new Notion database. Call it "Client CRM". Add the following properties:

| Property | Type | Purpose |
|---|---|---|
| Client Name | Title | Primary identifier |
| Phone | Phone | For SMS follow-ups |
| Email | Email | For email campaigns |
| Last Visit | Date | Triggers follow-up logic |
| Services | Multi-select | What they get done |
| Preferences | Text | Allergy notes, style preferences, etc. |
| Staff Member | Select | Who they prefer to work with |
| Next Follow-Up | Date | Auto-calculated by Claude |
| Lifetime Value | Number | Running total of spend |
| Birthday | Date | For birthday campaigns |
| Notes | Text | Anything else Claude learns |
| Status | Select | Active, Lapsed, VIP |

Here's the JSON structure Claude uses when creating or updating a client record via the Notion MCP:

```json
{
  "parent": { "database_id": "YOUR_DATABASE_ID" },
  "properties": {
    "Client Name": {
      "title": [{ "text": { "content": "Sarah Chen" } }]
    },
    "Last Visit": {
      "date": { "start": "2026-06-13" }
    },
    "Services": {
      "multi_select": [
        { "name": "Full Highlights" },
        { "name": "Gloss Treatment" }
      ]
    },
    "Next Follow-Up": {
      "date": { "start": "2026-08-08" }
    },
    "Lifetime Value": {
      "number": 1240
    },
    "Notes": {
      "rich_text": [{ "text": { "content": "Prefers ash tones. Sensitive scalp — use low-ammonia developer. Mentioned she has a wedding in September." } }]
    },
    "Status": {
      "select": { "name": "VIP" }
    }
  }
}
```

## Installing the Notion MCP Server

First, install the Notion MCP server package:

```bash
npm install -g @notionhq/mcp-server-notion
```

Then add it to your Claude Desktop config file. On Mac, that's `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/mcp-server-notion"],
      "env": {
        "NOTION_API_KEY": "secret_your_notion_integration_token_here"
      }
    }
  }
}
```

To get your Notion API key: go to [notion.so/my-integrations](https://www.notion.so/my-integrations), create a new integration, give it read/write access to your CRM database, and copy the secret token. Then share your CRM database with the integration inside Notion (click "Share" on the database → add your integration by name).

Restart Claude Desktop. You now have a Claude that can read and write to your Notion CRM.

## The 4 Automation Workflows

### Workflow 1: After-Appointment Log

Every time an appointment is marked complete, Claude reads the appointment notes and updates the client record. The appointment data might come from a webhook, a CSV export, or even a voice memo you transcribed.

**Claude prompt template:**

> "Here are the appointment notes from today's visit with [client name]: [notes]. Update their Notion CRM record with: last visit date, services performed, any new preferences or notes mentioned, updated lifetime value (add $[amount] to their current total), and set next follow-up date based on the service type."

Claude handles the Notion MCP calls automatically — it knows to search for the existing record first, then update rather than create a duplicate.

### Workflow 2: Follow-Up Scheduler

Different services have different return cadences. Claude applies these rules automatically:

| Service | Follow-Up Window |
|---|---|
| Haircut | 4–6 weeks |
| Full highlights | 7–9 weeks |
| Dog grooming | 6–8 weeks |
| Gym membership check-in | 2 weeks |
| Facial/skin treatment | 4 weeks |
| Lash extensions | 3 weeks |
| Deep conditioning | 8 weeks |

When Claude updates the "Last Visit" field, it simultaneously calculates and sets "Next Follow-Up" based on what services were performed. Three days before that date, a separate scheduled script queries Notion for upcoming follow-ups and drafts personalized outreach messages.

### Workflow 3: Birthday and Anniversary Reminders

Each morning, a scheduled Python script (running via cron or a free service like Railway) queries the Notion database for clients whose birthdays fall within the next 7 days:

```python
from notion_client import Client
from datetime import datetime, timedelta

notion = Client(auth="secret_your_token")

def get_upcoming_birthdays(database_id: str, days_ahead: int = 7):
    today = datetime.now()
    target_date = today + timedelta(days=days_ahead)
    
    response = notion.databases.query(
        database_id=database_id,
        filter={
            "and": [
                {
                    "property": "Birthday",
                    "date": {
                        "is_not_empty": True
                    }
                },
                {
                    "property": "Status",
                    "select": {
                        "does_not_equal": "Lapsed"
                    }
                }
            ]
        }
    )
    
    upcoming = []
    for page in response["results"]:
        birthday_prop = page["properties"].get("Birthday", {})
        if birthday_prop.get("date"):
            birthday_str = birthday_prop["date"]["start"]
            birthday = datetime.strptime(birthday_str, "%Y-%m-%d")
            # Compare month/day only (ignore year)
            birthday_this_year = birthday.replace(year=today.year)
            if today.date() <= birthday_this_year.date() <= target_date.date():
                name = page["properties"]["Client Name"]["title"][0]["text"]["content"]
                upcoming.append({
                    "name": name,
                    "birthday": birthday_this_year.strftime("%B %d"),
                    "page_id": page["id"]
                })
    
    return upcoming

birthdays = get_upcoming_birthdays("YOUR_DATABASE_ID")
for client in birthdays:
    print(f"Draft birthday message for {client['name']} — birthday {client['birthday']}")
```

Claude then drafts a personalized birthday message for each client, which you review and send via your preferred channel (email, SMS, or WhatsApp).

### Workflow 4: Monthly Revenue Report

On the first of each month, Claude queries the entire Notion database and generates a revenue summary:

**Claude prompt:**

> "Query my Notion CRM database and generate a monthly revenue report including: total client visits in the past 30 days, revenue by service category, top 10 clients by spend, clients who haven't visited in 60+ days (lapsed risk), and projected revenue for next month based on scheduled follow-ups."

Claude reads the Notion data through MCP, runs the calculations, and returns a clean summary you can share with your team or use for planning.

## The Full Appointment-to-CRM Pipeline

Here's the complete Python script that connects your booking webhook to the Claude + Notion pipeline:

```python
from flask import Flask, request, jsonify
from anthropic import Anthropic
import json

app = Flask(__name__)
client = Anthropic()

SYSTEM_PROMPT = """You are a CRM assistant for a local service business. 
When given appointment data, you will:
1. Search the Notion database for the existing client record
2. Update all relevant fields (last visit, services, notes, lifetime value, next follow-up)
3. Create a new record if the client doesn't exist yet
4. Set next follow-up date based on service type rules
5. Flag VIP status for clients with lifetime value over $500

Use the Notion MCP tools to read and write data. Always confirm what you updated."""

@app.route("/webhook/appointment-complete", methods=["POST"])
def appointment_complete():
    data = request.json
    
    # Build appointment summary from webhook payload
    appointment_summary = f"""
    Client: {data.get('client_name')}
    Phone: {data.get('client_phone')}
    Email: {data.get('client_email')}
    Date: {data.get('appointment_date')}
    Services: {', '.join(data.get('services', []))}
    Amount charged: ${data.get('amount', 0)}
    Staff member: {data.get('staff_name')}
    Notes from appointment: {data.get('notes', 'None')}
    """
    
    # Send to Claude with Notion MCP access
    response = client.messages.create(
        model="claude-sonnet-4-5",
        max_tokens=2048,
        system=SYSTEM_PROMPT,
        messages=[
            {
                "role": "user",
                "content": f"Process this completed appointment and update the Notion CRM:\n\n{appointment_summary}\n\nNotion database ID: YOUR_DATABASE_ID"
            }
        ]
    )
    
    return jsonify({
        "status": "processed",
        "crm_update": response.content[0].text
    })

if __name__ == "__main__":
    app.run(port=5000)
```

Deploy this on Railway, Render, or even a $5 DigitalOcean droplet. Point your booking software's webhook URL at `/webhook/appointment-complete` and the pipeline runs automatically every time an appointment completes.

## FAQ

**Is Notion CRM good for small business?**

Yes — Notion is one of the best CRM foundations for small businesses specifically because it's infinitely flexible without being technically complex. You're not forced into a "deal pipeline" model designed for B2B sales teams. You design the database to match how your business actually works: by appointment, by service type, by client tier. The free Notion plan supports unlimited database entries, up to 10 guests (staff members), and full API access. For a 1–5 person business, you will not hit the free plan's limits.

**Does MCP work with the Notion free plan?**

Yes. The Notion API (which MCP uses under the hood) is available on all Notion plans including free. The only requirement is creating an integration at notion.so/my-integrations and sharing your database with it. There are no API call limits that a local business would realistically hit — the free tier allows thousands of API calls per month.

**How much does this cost to run per month?**

The cost breakdown for a typical local business (50–200 clients, daily appointment processing): Notion free plan ($0), Claude API at roughly $0.003 per appointment processed via claude-sonnet-4-5 (so $0.15–$0.60/month for 50–200 appointments), and a basic server to run the Python webhook ($0 on Railway's free tier or $5/month on a small droplet). Total: **$0–$6/month**. Compare that to $90+/month for Vagaro with CRM features, and the ROI calculation is instant.

**Can multiple staff members update the CRM?**

Yes. Notion's sharing model lets you invite team members to the CRM database with different permission levels. You can give front desk staff "edit" access (can update client notes after appointments), stylists "comment" access (can add notes but not delete records), and keep yourself as the full admin. The AI pipeline runs independently of user permissions — it uses the integration token which always has write access. Staff can also view and manually edit records in the Notion interface without touching any code.

**How do I migrate from spreadsheets to Notion CRM?**

The cleanest path: export your spreadsheet to CSV, then use Notion's built-in CSV import (click "+ New page" → "Import" → "CSV"). Notion will map your columns to database properties. After the initial import, run a one-time Claude prompt asking it to "review all records in my Notion CRM, identify missing fields, and flag any data quality issues." Claude will scan the entire database through MCP and give you a prioritized cleanup list. Budget about 2 hours for the initial migration and cleanup — it's a one-time cost that pays off in months of automated operation.
