## What Vapi Is — And Why It Is Different From a Simple Chatbot

[Vapi](https://vapi.ai) is a voice AI platform built specifically for real-time phone conversations. It is not text-to-speech layered over a script. Vapi handles the full complexity of actual phone calls: natural pauses, interruptions mid-sentence, background noise, people who change their minds halfway through a question, and callers who say "actually, wait" and then restart.

Under the hood, Vapi orchestrates three components in under 600ms:

- **Speech-to-text** (transcription of what the caller says in real time)
- **LLM reasoning** (Claude, GPT-4o, or Llama 3 — your choice of model)
- **Text-to-speech** (voices from ElevenLabs, Deepgram, or Cartesia)

The result is a phone agent that sounds remarkably close to a calm, knowledgeable human receptionist — one who never gets frustrated, never puts callers on hold, and works at 2 AM with the same energy as noon. For local businesses where the phone is still the primary booking channel, this is transformative.

### Step 3: Write Your System Prompt Carefully

The system prompt is the single most important configuration element. A weak prompt produces a frustrating caller experience. A well-crafted one makes the AI indistinguishable from a trained receptionist.

**Always include in your prompt:**

1. **Business identity**: Name, address, exact hours — including every exception (closed Tuesdays, different Sunday hours, holiday closures)
2. **Complete service menu with prices**: The AI should never say "I am not sure about pricing" — callers ask this on almost every call
3. **Staff names and specialties**: Callers often ask for specific people by name or by specialty
4. **Booking rules**: Deposit requirements, cancellation policy, how far in advance you book
5. **Fallback instruction**: What the AI should say when a question falls outside its knowledge
6. **Tone guidance**: "Warm and casual like a friendly coworker" vs. "Professional but approachable" produces noticeably different output

**What to avoid:**

- Vague instructions like "be helpful" (too generic to guide behavior)
- Leaving out your cancellation policy (callers will ask about it)
- Forgetting to tell the AI it must use the check_availability tool rather than guessing

### Step 4: Get a Phone Number and Set Up Call Routing

You have two practical options for getting calls to your Vapi agent:

**Option A — Buy a Vapi number ($2/month)**

In the Vapi dashboard, go to **Phone Numbers → Buy Number**. Choose a local area code that matches your city. This is the simplest path for new businesses or when adding a dedicated line.

**Option B — Forward your existing business number (free)**

Most phone carriers support conditional call forwarding at the carrier level — no app needed. When your line is unanswered after 3–4 rings, or busy, the call silently forwards to your Vapi number. Your existing customers keep calling the same number they know. You pick up during business hours; Vapi catches everything else.

Forwarding codes by carrier (US):

```bash

# Replace XXXXXXXXXX with your 10-digit Vapi number
# Dial from your business phone to activate; dial the deactivation code to disable
```

For VoIP services (Google Voice, RingCentral, Grasshopper), the forwarding rules are in the account settings dashboard — typically under "Call Forwarding" or "Call Flow."

### Step 5: Build the Webhook Server

When Vapi needs to check availability or create a booking, it sends an HTTP POST request to your webhook URL with the function name and parameters. Your server processes the request and returns a natural-language result that the AI reads back to the caller.

Here is a complete Python webhook server using FastAPI:

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
import sqlite3
from datetime import datetime
import httpx
import os

app = FastAPI()

DB_PATH = "/var/data/salon_bookings.db"

def get_db():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    return conn

@app.post("/webhook/vapi")
async def vapi_webhook(request: Request):
    body = await request.json()
    message = body.get("message", {})
    msg_type = message.get("type")

    if msg_type == "function-call":
        function_name = message["functionCall"]["name"]
        params = message["functionCall"]["parameters"]

        if function_name == "check_availability":
            return await handle_check_availability(params)
        elif function_name == "create_booking":
            return await handle_create_booking(params)

    # Handle other Vapi message types (call started, ended, transcript, etc.)
    if msg_type == "end-of-call-report":
        await log_call_summary(message)

    return JSONResponse({"status": "ok"})

async def handle_check_availability(params: dict):
    date_str = params.get("date")
    stylist = params.get("stylist", "any")
    service = params.get("service_type", "")

    db = get_db()
    try:
        if stylist and stylist.lower() != "any":
            slots = db.execute("""
                SELECT time_slot, stylist_name
                FROM available_slots
                WHERE date = ? AND stylist_name LIKE ? AND is_booked = 0
                ORDER BY time_slot
                LIMIT 5
            """, [date_str, f"%{stylist}%"]).fetchall()
        else:
            slots = db.execute("""
                SELECT time_slot, stylist_name
                FROM available_slots
                WHERE date = ? AND is_booked = 0
                ORDER BY time_slot
                LIMIT 5
            """, [date_str]).fetchall()

        if not slots:
            return JSONResponse({
                "result": f"I don't see any openings on {date_str} for that service. Would you like me to check a different date, or I can put you on our cancellation list?"
            })

        slot_list = ", ".join([
            f"{row['time_slot']} with {row['stylist_name']}"
            for row in slots
        ])
        return JSONResponse({
            "result": f"Great news — on {date_str} we have: {slot_list}. Which of those works best for you?"
        })
    finally:
        db.close()

async def handle_create_booking(params: dict):
    db = get_db()
    try:
        db.execute("""
            INSERT INTO bookings
              (client_name, client_phone, date, time_slot, stylist, service, created_at, source)
            VALUES (?, ?, ?, ?, ?, ?, ?, 'vapi_phone')
        """, [
            params["client_name"],
            params["client_phone"],
            params["date"],
            params["time_slot"],
            params.get("stylist", "TBD"),
            params["service_type"],
            datetime.utcnow().isoformat()
        ])

        # Mark the slot as booked so it cannot be double-booked
        db.execute("""
            UPDATE available_slots
            SET is_booked = 1
            WHERE date = ? AND time_slot = ? AND stylist_name = ?
        """, [params["date"], params["time_slot"], params.get("stylist", "TBD")])

        db.commit()

        # Send SMS confirmation via Twilio
        await send_sms_confirmation(params)

        return JSONResponse({
            "result": (
                f"You're all set, {params['client_name']}! "
                f"I've booked your {params['service_type']} appointment on {params['date']} "
                f"at {params['time_slot']}. "
                f"You'll receive a confirmation text at {params['client_phone']} shortly."
            )
        })
    except Exception as e:
        db.rollback()
        return JSONResponse({
            "result": (
                "I ran into a small issue completing that booking on my end. "
                "Let me have one of our team members call you back within the hour to confirm. "
                "Can I verify I have the right number for you?"
            )
        })
    finally:
        db.close()

async def send_sms_confirmation(booking: dict):
    """Send an SMS booking confirmation via Twilio."""
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"https://api.twilio.com/2010-04-01/Accounts/YOUR_TWILIO_ACCOUNT_SID/Messages.json",
            auth=("YOUR_TWILIO_ACCOUNT_SID", "YOUR_TWILIO_AUTH_TOKEN"),
            data={
                "From": "+1YOUR_TWILIO_PHONE_NUMBER",
                "To": booking["client_phone"],
                "Body": (
                    f"Hi {booking['client_name']}! Your {booking['service_type']} appointment "
                    f"at Bella Hair Studio is confirmed for {booking['date']} at {booking['time_slot']}. "
                    f"Reply CANCEL to cancel (24hr notice required). See you soon!"
                )
            }
        )
    return response

async def log_call_summary(message: dict):
    """Log call metadata for analytics."""
    db = get_db()
    try:
        db.execute("""
            INSERT INTO call_logs (call_id, duration_seconds, ended_reason, transcript_summary, created_at)
            VALUES (?, ?, ?, ?, ?)
        """, [
            message.get("call", {}).get("id"),
            message.get("durationSeconds", 0),
            message.get("endedReason"),
            message.get("summary", ""),
            datetime.utcnow().isoformat()
        ])
        db.commit()
    finally:
        db.close()
```

Deploy this server on Railway, Render, or a small VPS. The webhook URL you enter in the Vapi assistant config should be your public HTTPS endpoint — for example, `https://salon-webhook.yourdomain.com/webhook/vapi`.

## Common Mistakes to Avoid

**1. Writing a system prompt so restrictive the AI says "I don't know" constantly**

Spend one hour writing out every question you have ever received over the phone — hours, parking, "do you take walk-ins," product questions, gift card questions. Put the answers directly in the prompt. Your AI should be able to handle 90% of calls without reaching a fallback.

**2. No human fallback path**

Always include a fallback instruction: "If a caller is upset, asking for the owner, or asking something you genuinely cannot answer, tell them you will have someone call them back within the hour." Then actually call them back within the hour. The fallback builds trust and saves relationships.

**3. Skipping SMS confirmations**

A confirmation text reduces no-shows by approximately 30% in most salons. Wire up Twilio or a similar provider in your webhook from day one — do not treat it as optional.

**4. Testing only in the browser console**

The Vapi dashboard has a test widget, but the experience over an actual phone call on a smartphone sounds different. Background noise, cellular compression, and real speech patterns all affect performance. Make 10 real test calls from your cell before going live.

**5. Not reviewing the call transcripts**

Vapi logs full transcripts of every call. Review them weekly for the first month. You will find common questions your prompt does not answer, phrasing patterns that confuse the AI, and opportunities to improve your service menu or booking flow.

## FAQ

**How human-sounding is Vapi, really?**

With a high-quality ElevenLabs voice (Rachel, Bella, and Domi are consistently rated as the most natural) and a well-written system prompt, most first-time callers cannot tell they are speaking with an AI. The main tells are a brief pause (1–2 seconds) when the agent is looking up information via a tool call, and the absence of the small verbal stumbles humans make. Callers who call back a second time and remember the previous interaction sometimes ask "is this a robot?" — at which point the AI should acknowledge it is a virtual assistant while staying helpful. For booking, FAQ, and hours calls, the experience is genuinely good. For emotionally charged conversations, the human fallback matters.

**Can Vapi handle callers with strong accents or regional dialects?**

Yes, better than most alternatives. Deepgram's Nova-2 model — the transcriber used in the config above — was trained on a highly diverse dataset including non-native English speakers, Southern US drawls, New York accents, and international callers. In practice, it handles most accents well. Callers who speak very quickly, have strong background noise, or use heavy technical jargon occasionally get mis-transcribed, which produces a slightly confused AI response. The fix is backchannel support (enabled in the config above), which lets the AI say "sorry, could you repeat that?" when it is uncertain rather than guessing.

**What happens when Vapi genuinely cannot answer a question?**

The system prompt's fallback instruction handles this. The AI says something like: "That is a great question — I want to make sure you get the right answer on that. Can I have one of our team members call you back within the hour?" The caller's contact info is already in the conversation, so your webhook logs it. Your job is then to actually make that callback within the promised window. This pattern — AI captures the lead, human closes — works better than either channel alone.

**How do I forward my existing business number to Vapi?**

The most robust method is conditional call forwarding set directly at the carrier level. Your existing number rings as normal. If unanswered after 3–4 rings, or if your line is busy, the call is silently redirected to your Vapi number. Customers keep calling the number they know. The carrier-level codes are in Step 4 above. If you use a VoIP provider (Google Voice, RingCentral, Grasshopper, OpenPhone), the forwarding rules are in the account dashboard under "Call Forwarding" or "Call Flow" — no codes needed. Conditional forwarding is available on virtually every US carrier with no additional monthly fee.

**Can Vapi send SMS confirmations after booking?**

Vapi itself handles only voice. SMS confirmations are sent by your webhook server as a downstream action. In the Python code above, after the `create_booking` function writes to the database, it fires a Twilio API call to send the confirmation text to the client's number. You can also trigger additional actions from the same webhook: add the booking to Google Calendar, create a CRM record, send a Slack notification to yourself, or initiate a follow-up reminder 24 hours before the appointment. The webhook is the integration hub — Vapi just triggers it.
